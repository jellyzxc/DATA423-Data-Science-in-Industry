Quality control using Xbar and S charts

Xiaocui Zhang 16142030

1 About the data

The file monitor.csv contains comma separated data. The columns are:
1. Timestamp - the timestamp of a model prediction being run
2. ProcessMemory - the allocated memory (MB) of the relevant server process
3. Prediction - the value predicted by the model
4. PredictionTimeMS - the duration of the prediction task in milliseconds

2 Data loading

4 Control charts generation

a) Is the memory usage of the server in control?

library(qicharts2)
pxbar_memory <- qicharts2::qic(x = Date, y = ProcessMemory, data = monitor.df, chart = "xbar", freeze = 20,  title   = 'Average allocated memory (MB) (Xbar chart)', 
                               ylab = 'the allocated memory (MB)',xlab = 'Date')
#Assume the first 20 days of data can be used to establish the control limits for the remainder of the data  => freeze = 20

plot(pxbar_memory)

summary(pxbar_memory)

According to the Xbar chart and the summary, there are 9 breaches(red points) of the upper and lower control limits as sigma.signal=9. The limit rule has been breached.
The runs.signal is 1 because longest.run (39)> longest.run.max(9) or n.crossings(18)<n.crossings.min(37), which indicates the runs rule and crossing rule have been breached.
The memory usage of the server is not in-control with respect to the Xbar chart. 

ps_memory <- qicharts2::qic(x = Date, y = ProcessMemory, data = monitor.df, chart = "s", freeze = 20,  title   = 'Standard deviation of allocated memory (MB) (S chart)',
                             ylab = 'the allocated memory (MB)',xlab = 'Date')
plot(ps_memory)

summary(ps_memory)

According to the S chart and the summary, there is no breaches(red points).The sigma.signal is 0 and runs.signal is 0,which indicates no rules are breached.
The memory usage of the server is in-control with respect to the S chart.

Since the Xbar chart is out of control, we conclude that the memory usage of the server overall is out of control.

pxbar_predictiontime <- qicharts2::qic(x = Date, y = PredictionTimeMS, data = monitor.df, chart = "xbar", freeze = 20,  title   = 'The mean of the duration of the prediction task in milliseconds (Xbar chart)',ylab = 'the prediction time of the model (ms)',xlab = 'Date')
plot(pxbar_predictiontime)

summary(pxbar_predictiontime)

According to the Xbar chart and the summary, there is no out-of-control points(red points).The sigma.signal is 0 and runs.signal is 0,which indicates no rules are breached.
The prediction time of the model is in-control with respect to the Xbar chart. 

ps_predictiontime <- qicharts2::qic(x = Date, y = PredictionTimeMS, data = monitor.df, chart = "s", freeze = 20,  title   = 'Standard deviation of the duration of the prediction task in milliseconds (S chart)',ylab = 'the prediction time of the model (ms)',xlab = 'Date')
plot(ps_predictiontime)

summary(ps_predictiontime)

According to the S chart and the summary, there are 2 breaches(red points) of the upper and lower control limits as sigma.signal=2. The limit rule has been breached.
The runs.signal is 1 because longest.run (11)> longest.run.max(9) or n.crossings(31)<n.crossings.min(37), which indicates the runs rule and crossing rule have been breached.
The prediction time is out of control in terms of the s chart.

As the S chart is out of control, the prediction time of the model is out of control in summary.

c) Is the stream of predictions in control?

pxbar_predictedValue <- qicharts2::qic(x = Date, y = Prediction, data = monitor.df, chart = "xbar", freeze = 20,  title   = 'The mean of the value predicted by the model (Xbar chart)',ylab = 'the value predicted by the model',xlab = 'Date')
plot(pxbar_predictedValue)

summary(pxbar_predictedValue)

According to the Xbar chart and the summary, there is no out-of-control points.The sigma.signal is 0 and the runs.signal is 0,which indicates no rules are breached.
The value predicted by the mode is in control according to the Xbar chart. 

ps_predictedValue <- qicharts2::qic(x = Date, y = Prediction, data = monitor.df, chart = "s", freeze = 20,  title   = 'Standard deviation of the value predicted by the model (S chart)',ylab = 'the value predicted by the model',xlab = 'Date')
plot(ps_predictedValue)

summary(ps_predictedValue)

According to the S chart and the summary, there are no red points and the sigma.signal=0. The limit rule has not been breached.
Howerver, the runs.signal is 1 because longest.run (35)> longest.run.max(9) or n.crossings(16)<n.crossings.min(37), which indicates the runs rule and crossing rule have been breached. There are 35 points that are consecutively below the middle line between Feb and the early March. The process Standard deviation was not in control during that time. And after then, the opints were consecutively above the middle line. The process is still not back in control.
Hence,the stream of predictions is out of control based on the s chart.

As the S chart is out of control, we conclude the stream of predictions overall is out of control.
